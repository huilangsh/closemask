# CloseMask 项目介绍

## 什么是 CloseMask？

CloseMask 是一个轻量级的 AI Agent 中间件代理，专为解决使用第三方 LLM API 时的隐私安全问题而设计。它位于 AI Agent 和第三方 LLM 服务之间，自动检测、遮罩和还原个人身份信息（PII）与技术凭证，确保用户敏感数据不会暴露给第三方 LLM 服务商。

### 为什么需要 CloseMask？

当你通过第三方中转 API 调用 LLM 时，所有请求数据都会离开你的基础设施，发送到第三方服务器。这意味着：

- **凭据泄露风险**：开发者调试时可能将 API Key、数据库密码等技术凭据发送给 LLM
- **用户隐私暴露**：客服对话中的手机号、身份证号、银行卡号等 PII 被传送到第三方
- **企业数据外泄**：员工在使用 AI 助手时不经意间分享了包含敏感信息的代码或配置
- **合规违规**：PII 数据的跨境传输可能违反 GDPR、PIPL、CCPA 等数据保护法规

CloseMask 在你和第三方 API 之间建立一道防护屏障——所有敏感数据在离开你的基础设施之前被替换为占位符，LLM 只能看到脱敏后的数据，而响应中的占位符会被自动还原为原始值。整个过程对用户和 Agent 完全透明。

## 核心价值

### 1. 隐私合规

在当今数据保护法规日益严格的环境下，企业面临着巨大的合规压力：

- **GDPR（通用数据保护条例）**：欧盟数据保护法规，要求数据主体有权控制其个人数据
- **CCPA（加州消费者隐私法案）**：美国加州隐私法规，赋予消费者更多数据控制权
- **PIPL（个人信息保护法）**：中国个人信息保护法，对个人信息的处理提出严格要求

CloseMask 通过在数据发送到 LLM 之前自动遮罩 PII，帮助企业满足这些法规要求，避免数据泄露和合规风险。

### 2. 零感知体验

CloseMask 对终端用户和开发者完全透明：

- **对用户**：无需改变使用习惯，PII 遮罩和还原完全在后台进行
- **对开发者**：无需修改现有代码，只需将 API 端点切换到 CloseMask 即可
- **对 Agent**：保持完整功能，包括工具调用、流式响应、多轮对话

### 3. 高性能

CloseMask 经过精心优化，可以处理高并发场景：

- **亚微秒级会话操作**：< 1μs 的会话查询和更新
- **低延迟处理**：10-50ms 的 PII 遮罩延迟
- **高吞吐量**：支持 1000+ 请求/秒
- **低资源占用**：< 50MB 内存占用

## 技术架构

### 双层架构设计

CloseMask 采用创新的双层架构，将 PII 检测与 Agent 功能分离：

```
┌─────────────────────────────────────────────────┐
│              Agent Layer (CloseMask)             │
│  - SSE 流式支持                                   │
│  - 工具调用处理                                   │
│  - 多轮对话持久化                                 │
│  - 会话管理                                       │
└───────────────────┬─────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────┐
│          PII Detection Layer (OneAIFW)            │
│  - 21+ PII 类型检测                               │
│  - 99%+ 准确率                                    │
│  - 中英文支持                                      │
│  - Zig+Rust 核心引擎                              │
└─────────────────────────────────────────────────┘
```

这种设计带来了以下优势：

1. **模块化**：可以独立升级 PII 检测引擎
2. **可扩展**：易于集成其他 PII 检测服务
3. **性能优化**：专门的 Agent 层针对 Agent 场景优化

### 关键技术选型

| 技术栈 | 说明 | 优势 |
|--------|------|------|
| **Go 语言** | 核心代理实现 | 高性能、并发友好、编译型语言 |
| **OneAIFW** | PII 检测引擎 | MIT 许可证、高准确率、轻量级 |
| **Redis** | 分布式会话存储 | 高性能、支持集群、持久化 |
| **Prometheus** | 指标收集 | 业界标准、生态丰富 |
| **SSE** | 流式响应协议 | HTTP 友好、低延迟 |

## 核心功能

## 使用场景

### 1. 客户服务 Agent

**场景描述**：企业客服系统使用 AI Agent 处理用户咨询，用户会提供身份证号、手机号等敏感信息。

**CloseMask 解决方案**：
```
用户："我的身份证号是 110101199003077777，帮我重置密码"
    ↓
CloseMask 遮罩身份证："我的身份证号是 ${ID_CARD_xxx}，帮我重置密码"
    ↓
LLM 处理遮罩后的请求
    ↓
CloseMask 还原响应："好的，已为您重置密码"
    ↓
用户收到正常回复
```

**价值**：保护用户隐私，满足合规要求

### 2. 金融助手

**场景描述**：银行或金融机构使用 AI Agent 提供财务建议，用户会提供银行账号、交易金额等敏感信息。

**CloseMask 解决方案**：
```
用户："转账 5000 元到账户 6222000012345678"
    ↓
CloseMask 遮罩账号和金额："转账 ${AMOUNT_xxx} 元到账户 ${BANK_CARD_xxx}"

LLM 处理遮罩后的请求
    ↓
CloseMask 还原响应："已成功转账 5000 元"
```

**价值**：避免敏感金融数据泄露，符合金融监管要求

### 3. 医疗应用

**场景描述**：医院或医疗机构使用 AI Agent 辅助医生诊断，会涉及患者姓名、病历号等敏感信息。

### 为什么选择 OneAIFW？

1. **MIT 许可证**：允许商业使用和修改

4. **持续维护**：活跃的开源社区
5. **易于集成**：提供 RESTful API

### OneAIFW 的局限性

OneAIFW 是一个通用的 PII 检测引擎，专注于文本中的 PII 识别和遮罩。它不直接支持：

- Agent 工具调用场景（tool_calls）
- SSE 流式响应代理
- OCR 等工具返回的 PII 处理

CloseMask 的 Agent 层正是为了填补这些空白而设计的。

## 快速开始

### 前置条件

- Go 1.21+
- LLM API（OpenAI、Anthropic、Azure 等）

> OneAIFW 为可选依赖，不部署时 CloseMask 使用内置检测（手机号/身份证/邮箱/银行卡/API Key），开箱即用。

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/huilangsh/closemask.git
cd closemask

# 2. 编译
go build -o closemask ./cmd/server

# 3. 配置
cp config.json.example config.json
# 编辑 config.json，至少填入 llm_url

# 4. 运行
./closemask
```

### 配置示例

```json
{
  "llm_url": "https://api.openai.com/v1",
  "port": 8846,
  "storage_type": "layered",
  "pii_engine": "auto",
  "local_mask_level": "strict"
}
```

### 验证安装

```bash
# 健康检查
curl http://localhost:8846/health

# 测试请求
curl -X POST http://localhost:8846/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [
      {"role": "user", "content": "我的电话是 13800138000"}
    ]
  }'
```
