## 推荐配置

### 场景 1：基础场景（零依赖）

仅使用内置正则检测，无需安装 Python 环境。

**适用场景**：
- 只需检测手机号、身份证、邮箱、银行卡、API Key 等结构化 PII
- 对性能要求高
- 不想引入额外依赖

**配置**：
```json
{
  "pii_engine": "builtin",
  "pii": {
    "ner_enabled": false
  }
}
```

**检测能力**：
- ✅ 手机号、身份证、邮箱、银行卡、IP 地址
- ✅ API Key、JWT Token、AWS Key、密码
- ❌ 人名、组织名、地址

### 场景 2：增强场景（内置 + NER）

内置正则 + NER 服务，检测能力最全面。

**适用场景**：
- 需要检测人名、组织名、地址等非结构化 PII
- 对隐私保护要求高
- 可接受额外依赖

**配置**：
```json
{
  "pii_engine": "auto",
  "pii": {
    "ner_enabled": true,
    "ner_mode": "remote",
    "ner_remote_endpoint": "http://127.0.0.1:8847"
  }
}
```

**检测能力**：
- ✅ 所有内置正则能检测的类型
- ✅ 人名、组织名、地址、日期等 NLP 实体

## 三层检测架构

CloseMask 采用三层检测架构，逐层增强：

```
┌─────────────────────────────────────────────────────────────┐
│                    第一层：LocalMasker                       │
│  检测：API Key / JWT / AWS Key / Bearer Token / 密码        │
│  特点：零依赖，毫秒级响应                                    │
├─────────────────────────────────────────────────────────────┤
│                    第二层：BuiltInPII                        │
│  检测：手机号 / 身份证 / 邮箱 / 银行卡 / IP 地址             │
│  特点：零依赖，毫秒级响应                                    │
├─────────────────────────────────────────────────────────────┤
│                    第三层：NER Service                       │
│  检测：人名 / 组织名 / 地址 / 日期                          │
│  特点：需要 Python 环境，响应时间 ~50ms                     │
└─────────────────────────────────────────────────────────────┘
```

## 测试结论

1. **内置正则适合检测结构化 PII**：手机号、身份证、邮箱等格式固定的 PII，正则匹配更精确、更快速。

2. **NER 服务适合检测非结构化 PII**：人名、组织名、地址等没有固定格式的 PII，需要 NLP 模型识别。

3. **两者互补**：内置正则 + NER 服务可以覆盖更全面的 PII 类型。

4. **性能差异**：内置正则 <1ms，NER 服务 ~56ms。对于大多数场景，这个延迟差异可以接受。
